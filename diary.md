### 🌐 WASM で遊んだ！

**Rust が ブラウザで動いた！！！**

```
Rust コード → wasm-pack → .wasm + .js → ブラウザで実行！
```

- `#[wasm_bindgen]` で JavaScript から呼べるようにする
- `wasm-pack build --target web` でブラウザ用にビルド
- JavaScript から Rust 関数を直接呼べる！

```rust
// Rust
#[wasm_bindgen]
pub fn add(a: i32, b: i32) -> i32 { a + b }
```

```javascript
// JavaScript
const result = add(2, 3);  // 5 (Rust で計算！)
```

**ブラウザで ML や ICN クライアントも動かせるかも！夢が広がる！**

### 学んだこと
**エラー処理:**
- `Result<T, E>` と `?` 演算子でエラー伝播
- `Box<dyn Error>` = なんでもエラー箱（多態性）
- `unwrap_or_else` vs `?` vs `if let Err` の使い分け
- **エラー処理は `main` に集約！** → 責務の分離

**リファクタリング:**
- `Config` 構造体でデータをまとめる
- `clone()` で所有権問題を回避
- `lib.rs` と `main.rs` の分離（別クレート扱い！）
- `minigrep::run` のように呼び出す必要がある

**ライフタイム実践:**
- `search<'a>(query: &str, contents: &'a str) -> Vec<&'a str>`
- **戻り値の参照元がどの引数か** を明示する必要がある
- `contents` から切り出した `&str` を返すから `'a` が必要

**環境変数:**
- `env::var("NAME")` で取得
- `is_err()` で「セットされていない」を判定

**標準エラー出力:**
- `eprintln!` でエラーだけ画面に表示
- リダイレクト時にエラーと結果を分離できる
